Transform given SQL symbol suppliers (like tables, views, etc.) into more convenient formats for string template literals.

const [execCtx, execHostID] = ec.qualifiedTokens( "my_schema1", (value, son) => ({ sqlInjection: son.injectable(value) }), pgd.execution_context, pgd.execution_host_identity, );

Once you have the above, you can use ${execCtx} and ${execHostID} in SQL template literals.

schemaObjectNames(...schemaNames: SchemaName[])

Return the qualified naming strategy for a given schema.

const [sQN, QN2] = ec.schemaQNI("my_schema1", "my_schema2");

After the call above, sQN will be the SqlObjectNames instance for my_schema1 namespace.

schemaQN(...schemaNames: SchemaName[]): [emit.SqlObjectNames, (text: string) => string][]

Return the qualified naming strategy for a given schema along with injectables wrapper to make it easy to create qualified references to SQL objects in SQL templates.

const [[sQN, sQR], [QN2, QR2]] = ec.schemaQNI("my_schema1", "my_schema2");

After the call above, sQN will be the SqlObjectNames instance for my_schema1 namespace and sQR will be a (text: string) => string function which will accept any text and qualify the reference as "my_schema1".${text}.

schemaQualifier(...schemaNames: SchemaName[])

Return a function for each schema that, when called, will "qualify" the given text in that schema in a format convenient for SQL template literals.

const [sQR, QR2] = ec.schemaQualifier("my_schema1", "my_schema2");

After the call above, sQR will be a (text: string) => SqlInjectable function which will accept any text and qualify the reference as "my_schema1".${text} in a method convenient for use within a SQL string template literal.
